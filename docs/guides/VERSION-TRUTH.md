# Version Truth

Single source of truth for every version number in the GWI POS system.

## Quick Reference

| Component | Authoritative Source | Example Value | Scheme |
|-----------|---------------------|---------------|--------|
| POS App | `version-contract.json` > `version` | `2.0.42` | semver |
| Schema | `version-contract.json` > `schemaVersion` | `124` | migration prefix (NNN) |
| Dashboard | `version-contract.json` > `dashboardVersion` | `0.3.1` | semver |
| Installer | `version-contract.json` > `installerVersion` | `2.0.42` | matches app version |
| Ansible Baseline | `installer/VERSION` | `baseline-2026.03.20.1` | `baseline-YYYY.MM.DD.N` |
| Docker Image Tag | `ghcr.io/getwithitman/gwi-pos:<version>` | `2.0.42` | matches app version |
| Git SHA | `version-contract.json` > `gitSha` | `02e67031` | 8-char abbreviated hash |

## Component Details

### POS App Version (`version`)

- **Source of truth:** `package.json` > `version`, published in `public/version-contract.json`
- **Generated by:** `scripts/generate-version-contract.mjs` at build time (reads `package.json`)
- **Bumped by:** `scripts/bump-version.sh` (auto: `1.1.<commit-count>`) or deploy pipeline sets `X.Y.Z` directly in `package.json`
- **Consumed by:** gwi-node.sh (deploy validation), MC (fleet tracking), NUC status API, sync-agent heartbeat

### Schema Version (`schemaVersion`)

- **Source of truth:** Highest `NNN-` prefix in `scripts/migrations/`
- **Generated by:** `scripts/generate-version-contract.mjs` (scans migration filenames at build time)
- **Consumed by:** NUC readiness gate, sync compatibility validation, MC rollout risk assessment
- **Also tracked:** `migrationCount` (total files), `schemaSha256` (hash of compiled schema), `schemaContractHash` (full introspected schema hash)

### Dashboard Version (`dashboardVersion`)

- **Source of truth:** `public/dashboard-version.txt` or extracted from bundled `public/gwi-nuc-dashboard.deb`
- **Built by:** `gwi-dashboard` repo CI, published as GitHub Release `.deb`
- **Installed by:** `gwi-node update_dashboard()` and `public/installer-modules/12-dashboard.sh`
- **Verified by:** `gwi-node dashboard-check` (compares installed dpkg version against contract)

### Installer Version (`installerVersion`)

- **Source of truth:** `version-contract.json` > `installerVersion` (always equals `version`)
- **Set by:** `scripts/generate-version-contract.mjs` (copies `appVersion`)
- **Consumed by:** `installer.run` self-update check against Vercel-hosted contract

### Ansible Baseline Version

- **Source of truth:** `installer/VERSION` (file content: `baseline-YYYY.MM.DD.N`)
- **Read by:** `scripts/generate-version-contract.mjs` from `installer/version.txt` fallback path, published as `ansibleBaselineVersion`
- **Scheme:** `baseline-YYYY.MM.DD.N` -- NOT semver, NOT the app version
- **Tracks:** hardening playbook version across 16 Ansible roles in `installer/site.yml`

### Docker Image Tag

- **Source of truth:** `ghcr.io/getwithitman/gwi-pos:<version>` where `<version>` matches `version-contract.json` > `version`
- **Built by:** GitHub Actions CI on push to main
- **Verified by:** Cosign keyless OIDC signature verification in `gwi-node.sh`
- **Consumed by:** `gwi-node deploy` (pulls and swaps container)

### Git SHA (`gitSha`)

- **Source of truth:** `git rev-parse --short HEAD` at build time
- **Published in:** `version-contract.json` > `gitSha` (8-char abbreviated hash)
- **Consumed by:** deploy verification, MC fleet status, NUC heartbeat

## How to Check a Running NUC

```bash
docker exec gwi-pos cat /app/public/version-contract.json
```

If you need to know what version a NUC should be running, read `version-contract.json` inside the running container. This is the single authoritative source for all component versions on every NUC.

## Rules

1. **One bump per deploy.** `version-contract.json` is regenerated every build. MC auto-detects version changes.
2. **Never hand-edit `version-contract.json`.** It is generated output. Edit `package.json` or `installer/VERSION` instead.
3. **Ansible baseline version is independent.** It follows `baseline-YYYY.MM.DD.N`, not app semver.
4. **Docker tag must match app version.** Any mismatch means the deploy pipeline is broken.
